SystemC‚Ñ¢ Library Release Notes

This document identifies important information for the SystemC‚Ñ¢ Library package for Simics 7. All users of this specific package should review this document carefully.

Changes Since Version 7.0.0

The following is a list of changes since the initial 7.0.0 release.

7.11.0 (build 7079)

SystemC Library

  • Adjust SC verbosity level according to Simics log level (fixes SIMICS-23083).

7.10.0 (build 7075)

7.9.0 (build 7071)

SystemC Library

  • Replaced offending words/phrases with inclusive language (fixes SIMICS-23059).

7.8.0 (build 7069)

SystemC Library

  • Enhanced the logic for determining AS_VERSION and LD_VERSION. The updated logic now identifies version patterns in the format X.Y.

7.7.0 (build 7064)

SystemC Library

  • Removed unused simics::iface::PciBusInterface class.

7.6.0 (build 7063)

SystemC Kernel

  • Resolved MSVC 2022 warning C4530 by adding /EHsc flag.

7.5.0 (build 7061)

SystemC Checkpoint Documentation

  • Added back the PDF documentation (fixes SIMICS-22949).

7.4.0 (build 7055)

SystemC Library

  • Added optional '-std'/'--cxx-standard' option to the build-systemc-libraries build script to enable building libs with a different C++ language standard. Current supported options are 14 and 17.

7.3.0 (build 7053)

SystemC Library

  • A clear error message is now printed when SYSTEMC_KERNEL_VERSION is not set.
  • Added support for SystemC kernel version 3.0.0. Current build options for SYSTEMC_KERNEL_VERSION are 2.3.3, 2.3.4 and '3.0.0'.

7.2.0 (build 7046)

SystemC Kernel

  • Added kernel 2.3.4.1_20240904-Intel. Use build option SYSTEMC_KERNEL_VERSION to select the kernel version.

SystemC Library

  • Moved some GNU Make definitions from Base into SystemC add-on package. SystemC users using Simics Base 6.0.209 or newer must upgrade 1013 add-on package to version 7.2.0.
  • The SystemC libraries have been reorganized and are now located in separate subdirectories based on their respective kernel versions. Specifically, you can find them under: <host>/lib/systemc/2.3.3 and <host>/lib/systemc/2.3.4
  • Added '--version' option to the build-systemc-libraries build script to enable building libs with a different version of SystemC kernel. The default version used is 2.3.3.
  • Updated the version check logic. The SCL_VERSION variable now uses the build-id of package 1013 (SystemC-Library).
  • All variables ending in _not_set in the class PcieTlmExtension have been renamed to end with _set to improve clarity and consistency. They now default to false, indicating that the corresponding variable is not set.
  • Extended the class PcieTlmExtension with ide_secured ATOM support.
  • Added support for selecting SystemC kernel versions. Current build options for SYSTEMC_KERNEL_VERSION are 2.3.3 and 2.3.4. Users of add-on package #7213 must upgrade to the latest version.
  • The C++ language standard C++14 (-std=c++14) is no longer used by default. Now the default C++ language standard is determined by the compiler, i.e, C++17 for GCC 12.

7.1.0 (build 7039)

SystemC Library

  • Extended get_device_id method of tlm2simics::pcie_map class with parameter dev_obj. This makes it easier to support PCIe ports.
