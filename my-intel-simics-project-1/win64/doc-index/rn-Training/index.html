Simics Training Release Notes

This document identifies important information for the Simics Training package for Simics 7. All users of this specific package should review this document carefully.

Changes Since Version 7.0.0

The following is a list of changes since the initial 7.0.0 release.

7.0.0-pre.11 (build 7078)

Sample Device for package priorities

  • Make package detection work regardless of use of ".." in package and module paths.

Simics Training Materials

  • Updates to Simics Internals Labs after train-the-trainer sessions.

7.0.0-pre.10 (build 7075)

7.0.0-pre.9 (build 7061)

Simics Training Materials

  • General content clean-up and move to use externally downloadable Yocto Linux instead of Clear Linux from the (no longer distributed) package 4094.

7.0.0-pre.8 (build 7058)

Simics Training Materials

  • Update package contents based on feedback from first beta training run.

7.0.0-pre.7 (build 7055)

Simics Training Materials

  • Simics Internals training lab documents added.

7.0.0-pre.6 (build 7028)

Simics Training Materials

  • Fixed an issue in the Intel Simics New User Training lab 003, part B, regarding the behavior of params.list -only-changed.
  • Fixed issue in the Intel Simics New User Training lab 003 (wrong output shown for a lookup-file command).
  • Fixed issue in the Intel Simics New User Training lab 003 (a bad wait-for string). Also some fixing some typos.
  • The code for the m-compute module was restructured to test all variants of the module implementation on every build (fixes SIMICS-21561). The Workshop 02 instructions were updated to reflect the new code structure.
  • Updated New User Training Lab 003 PDF.
  • The code for the m-control module was restructured to also test the stalling version on every build (SIMICS-21573). Workshop 02 instructions were updated to reflect the new structure.

7.0.0-pre.5 (build 7023)

Common

  • The signing certificate on Simics packages has been updated.

Common

  • Updated the batch (.bat) files in the bin folder so that they do not leak environment variables to the current cmd.exe command shell on Windows (fixes SIMICS-21906).
