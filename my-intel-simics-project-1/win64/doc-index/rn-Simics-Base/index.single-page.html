Intel® Simics® Simulator Release Notes

Changes Since Version 7.0.0

The following is a list of changes since the initial Intel® Simics® Simulator 7.0.0 release.

7.38.0 (build 7080)

Core

  • New API function, SIM_event_class_flags to get the registered event flags for an event class.
  • New API function, SIM_get_event_class to get hold of the event class registered on a configuration class.

Device Modeling Language (DML) 1.4

  • Improved the log messages that are output by default for partial register accesses (fixes #356).

Probe Monitors

  • The summary command has a new argument, histogram_max_lines, where the maximum lines in a histogram can be set, otherwise the max number of lines are taken from the table settings as given by the table-setting command in the monitor.

Simics Migration Guide

  • Adding import clarification to documentation (fixes SIMINT-1673).

7.37.0 (build 7079)

Blueprint framework

  • Add port object handling to blueprint component wrappers (SIMICS-23038).

C++ Device API v2

  • The attribute setter now provides an informative message to the user, rather than logging an error message on the device object.
  • The class method set_bank_as_initialized in MappableConfObject has been renamed and its functionality updated to write_protect_iface_maps. This change is designed to offer a clearer and more intuitive approach to managing the write protection status of iface_maps.

Core

  • A bug was fixed where the cli.sb_create Python function sometimes failed to start a script branch.
  • The set-min-latency and set-time-quantum commands now have the same count and unit arguments as the run command.

Project Setup Script

  • The generated project trampolines now use the value of the SIMICS_PYTHON environment variable that was set when project-setup was run.

  • Allow nesting SIM_process_work calls that have done callbacks (fixes SIMICS-16444).

  • Simics will now issue a warning if two modules with the same name are detected in the same directory. It is undefined which module will take precedence over the other.

  • The search order for Simics modules has been changed: If multiple modules with the same name are available on the search path, then Simics will select one based only on which directory it resides in. Most properties of the module search order remain the same as before; in particular, modules built in the project or added with add-module-directory still have higher priority than modules from any package, and modules from prioritized packages still have higher priority than modules from non-prioritized packages. However, when choosing among conflicting modules e.g. among non-prioritized packages, Simics previously preferred the module with the newest build date; this property is no longer significant.

    Technically, the new search order is based on Python's sys.path: Simics adds entries in sys.path for all module directories, and then searches for modules based on the order they appear there. If the Simics-managed sys.path entries are tampered with, some commands (set-prioritized-package, add-module-directory) will report an error and refuse to work. A new CLI command reset-sys-path has also been added to restore these sys.path entries to a consistent state.

New Features of Intel® Simics® Simulator 7

This section highlights the main new functionality in Intel® Simics® Simulator 7 compared to the current version. Most new features in Intel® Simics® Simulator are introduced in minor versions, while majors versions are used to remove deprecated functionality since breaking compatibility is not allowed in a minor version. Still, the first release of Intel® Simics® Simulator 7 has a number of new features and many more are planned to be introduced during the product's lifetime.

In-Memory Snapshots

With this feature a user can take a snapshot of the state of the complete simulated system in run-time and revert back to this state at any point in time later in the simulation. While the mechanism for taking snapshots already exists in Intel® Simics® Simulator as part of the reverse execution feature, it has never been an accessible feature by itself. In Intel® Simics® Simulator 7, in-memory snapshots will replace the reverse execution functionality.

Use Any Python

Intel® Simics® Simulator 7 can be run with any modern Python that the user has installed. Currently Python 3.10, 3.11, 3.12 and 3.13 are supported by Intel® Simics® Simulator 7, but the list of Python versions supported may change by the time of release. Python is still bundled with the product by default in the initial release of Intel® Simics® Simulator 7, but this is planned to change (see below). With this change, users have more control over when they change the major version of Python used in the simulator.

Intel® Simics® Simulator as a Python Module

Intel® Simics® Simulator will include a "simics" Python module that users can add to their existing Python installations using the standard pip tool. Using the simulator as a Python module greatly simplifies integrating Intel® Simics® Simulator in existing test environments for example.

CMake as Default

In Intel® Simics® Simulator it has been possible to build modules using CMake as the build tool for some time. Starting with version 7, CMake will become the default build tool. GNU Make will remain supported as a legacy feature.

Semantic Versioning of Simics-Base

The Intel® Simics® Simulator package versioning has been updated to comply with the industry standard semantic versioning scheme. The previous versioning scheme used by Intel® Simics® Simulator was very similar to semantic versioning, so it is a initially a small change. The biggest difference is that the minor version is now represented by the second digit group in the version number. I.e. the minor releases after Intel® Simics® Simulator 7.0.0 will be 7.1.0, 7.2.0, etc. The patch version will not be used initially.

Updated Host Requirements

The Windows version of Intel® Simics® Simulator now requires Windows 10.

Deprecations and End-of-life

API and script compatibility with Intel® Simics® Simulator 4.8 and older has been dropped. Refer to the Simics 7 Migration Guide for a complete list.
